About

Osseous

A bony CSS framework without guts.

Download View on GitHub Report Issue

What is it?

Osseous is a small CSS framework that only includes some (bony) elements to grow a website. Its meant to be a starting point that you can add additional features (guts) on top of, as you require them. Rather than importing a massive framework and not utilizing all of the features.

What's Included?

  • Typography
  • Columns, Rows, Grid
  • Tables
  • Buttons, Forms, Inputs
  • Navbar
  • Sidebar
  • Dropdowns

What's Missing?

  • Alerts
  • Cards
  • Carousel
  • Icons
  • Modals / Popovers
  • Tooltips
  • Animations

Examples

Osseous includes formatting for all the basic elements, and some common features like media cards, mobile menu, and navigation.

Use the examples below as a reference and demonstration of this bony framework.

Root Variables

The default fonts and colors are up top. In the global :root object for easy editing. The default font-family is -apple-system,BlinkMacSystemFont but you can change this by editing the root variable: --font-default. The default font-family for monospace elements is SFMono-Regular but again, you can change this by editing the root variable: --font-monospace. 

:root {
    --red: 239, 50, 91; /* RGB colors instead of HEX, so opacity can be applied. */
    --green: 50, 239, 135;
    --blue: 51, 195, 240;
    --yellow: 239, 239, 50;
    --grey: 255, 255, 255;
    --black: 34, 34, 34;

    --font-default:-apple-system,BlinkMacSystemFont,"Segoe UI",Roboto,"Helvetica Neue",Helvetica,Arial,sans-serif;
    --font-monospace:"SFMono-Regular",Consolas,"Liberation Mono",Menlo,"Courier New",monospace;
}

Buttons

Use the button class for styling anchors, inputs, buttons, to look like a button. And instead of confusing sub-button names like primary, success, info, warning, or danger... Osseous provides 3 simple colored buttons; red, green, and blue. Which you can also edit the default colors in the :root variables, to match your theme. The button-blue class could mark important actions and button-red could mark dangerous actions. But we're not going to tell you how you should design your site.

The nav-large-menu will hide on small screens. While the nav-small-menu will show up in its place.

In this example, the small menu just contains one element; A menu icon that opens the Sidebar.

Sidebar

Use the sidebar class for mobile navigation bar on the left side. But be sure to provide an id for the sidebar, so that it can easily be opened/closed using JavaScript. The default id provided is simply, "sidebar".

<nav id="sidebar" class="sidebar shadow-right">
    <div class="nav-item-close">
        <a class="u-right" title="Close Menu" href="javascript:closeSidebar()" aria-label="Close">
            <svg width="16" height="16" viewBox="0 0 16 16" xmlns="http://www.w3.org/2000/svg"><path d="M2.757 14.657L8 9.414l5.243 5.243 1.414-1.414L9.414 8l5.243-5.243-1.414-1.414L8 6.586 2.757 1.343 1.343 2.757 6.586 8l-5.243 5.243z" fill-rule="evenodd"></path></svg>
        </a>
    </div>

    <div class="nav-list text-center" onclick="closeSidebar()">
        <a class="nav-item" href="#link1">Link 1</a>
        <a class="nav-item" href="#link2">Link 2</a>
        <a class="nav-item" href="#link3">Link 3</a>
    </div>
</nav>

Showing the sidebar requires some JavaScript to work. But its only 2 lines worth.

function openSidebar(){document.getElementById("sidebar").style.display="block";}
function closeSidebar(){document.getElementById("sidebar").style.display="none";}
